### 1.2. Цели сайта

Сайт — **полноценный цифровой продукт**, а не визитка. Главная бизнес-цель: провести пользователя от первого визита до **подтверждённой записи** и **готовности к экзамену/занятию** с минимальным числом звонков.

**Задачи:**

- Информирование о программах, услугах, расписании и документах
- Онлайн-запись в тир и на курсы с управлением слотами
- Калькулятор предварительного расчёта стоимости
- Тренажёр-тест по правилам безопасного обращения с оружием (аналог экзамена Росгвардии)
- Демонстрация преимуществ школы и регионального УТП
- Соответствие законодательным требованиям к сайтам образовательных организаций
- Интеграция 3D-визуализации для повышения вовлечённости и доверия к профессионализму школы

**Не входит в v1–v2 (перенесено в v3):**

- Личный кабинет клиента (регистрация, история, документы, платежи)
- Авторизация через Госуслуги (ESIA)
- История платежей и загрузка документов клиентом

## 2. Стратегия релизов (v1 → v2 → v3)

Проект реализуется как **полноценный продукт** в три релиза. Каждый релиз — законченный, работающий сайт, готовый к эксплуатации.

**Срок:** недели 1–10  
**Результат:** запуск публичного сайта с полным контентом и базовой конверсией.

| Компонент | Описание |
|-----------|----------|
| Все публичные страницы | Главная, услуги (4 направления), контакты, `/docs/` |
| Дизайн-система | UI-kit, адаптив, версия для слабовидящих |

| Документы | Скачивание заявлений, договоров, прейскурантов (PDF/DOCX) |
| SEO-база | ЧПУ, meta, sitemap, robots, Яндекс.Метрика |

**Срок:** недели 11–22

## 4. Требования к дизайну и 3D-графике

### 4.1. Общий стиль

- **Современный минимализм** с акцентом на профессионализм и надёжность
- Строгость — в **типографике, сетке и структуре**, не в «военном» декоре (камуфляж, клише с перекрестием)
- Фотографии реального тира, инструкторов, занятий — приоритет над стоковыми иллюстрациями
- **Без hero-слайдера** на главной: статичный hero + блок «Ближайшие занятия»

### 4.2. Цветовая палитра

| Элемент | Цвет | Применение |
|---------|------|------------|
| Основной акцент | `#1A3A6B` | Хедер, CTA-секции, акцентные блоки |
| CTA-кнопки | `#1A3A6B` (primary), `#333333` (secondary) | Запись, скачивание |
| Текст | `#222222` | Основной контент |
| Фон | `#FFFFFF` / `#F5F5F5` | Контентные блоки |
| Успех / ошибка | `#2E7D32` / `#C62828` | Формы, тест |

### 4.3. Типографика и адаптив

- Не более **2 шрифтов** (рекомендация: **Inter** + **Montserrat** для заголовков)
- Основной текст: **16px+**, заголовки: **24px+**
- **Mobile-first**, breakpoints: `320`, `768`, `1024`, `1440` px
- Версия для слабовидящих: шрифт **18–20px**, высококонтрастный режим, `localStorage`, `prefers-reduced-motion`
- Анимации: только **микро** (hover, focus, переходы 150–250 ms). Без тяжёлых scroll-анимаций

### 4.4. 3D-графика — концепция и варианты реализации

3D используется **точечно**, для усиления доверия и вовлечённости, без ущерба производительности (см. 4.4.5).

#### 4.4.1. Сцены и размещение

| # | Сцена | Страница | Приоритет | Релиз |
|---|-------|----------|-----------|-------|
| 1 | **Hero: стрелок и мишень** (общий ракурс тира) | Главная | Высокий | v1 |
| 2 | **Интерактив: как человек стреляет в мишень** | Главная / «О школе» / «Секции» | Высокий | v2 |
| 3 | **3D-тур тира 25 м** (panorama / walkthrough) | «Секции» / «О школе» | Средний | v2 |
| 4 | **Мишенная установка** (разбор элементов) | «Секции», «Аренда» | Средний | v2 |
| 5 | **Поза / стойка стрелка** (обучающая) | «Обучение граждан» | Низкий | v2+ |

#### 4.4.2. Сценарий «Как человек стреляет в мишень» (ключевая сцена)

**Пользовательский сценарий:**

1. На экране — упрощённая 3D-сцена: стрелок (силуэт/стилизованная модель), линия огня, мишень на дистанции 25 м
2. Пользователь нажимает **«Показать технику»** или прокручивает до блока — запускается **пошаговая анимация**:
   - Шаг 1: стойка и хват
   - Шаг 2: прицеливание
   - Шаг 3: контроль дыхания / спуск
   - Шаг 4: попадание / группировка на мишени
3. Под каждым шагом — **краткая подпись** (1–2 предложения) от методики школы
4. На мобильных — упрощённая версия (2D fallback или короткий loop-видео WebM)

**Интерактив (v2):**

- Переключение шагов кнопками или свайпом
- Опционально: поворот камеры вокруг стрелка (OrbitControls, ограниченный угол)
- **Без** симуляции реального оружия и без «игровой стрельбы» — только **обучающая визуализация**

#### 4.4.3. Варианты технической реализации 3D

| Вариант | Стек | Плюсы | Минусы | Рекомендация |
|---------|------|-------|--------|--------------|
| **A. Three.js + React Three Fiber** | `@react-three/fiber`, `@react-three/drei`, GLTF/GLB | Полный контроль, интерактив, интеграция с Next.js, lazy load | Нужна 3D-модель (Blender), оптимизация полигонов | **Основной вариант для v2** |
| **B. Spline (spline.design)** | Embed или `@splinetool/react-spline` | Быстрый прототип, редактор без кода | Зависимость от Spline, лимиты экспорта | **Прототип v1 / hero v1** |
| **C. Lottie / Rive (2.5D)** | `lottie-react` / `@rive-app/react-canvas` | Лёгкий вес, отличный mobile, a11y | Не настоящий 3D, ограниченный orbit | **Fallback для слабых устройств** |
| **D. Pre-rendered video** | WebM/MP4 loop, `<video>` + poster | Максимальная совместимость | Нет интерактива | **Fallback + reduced-motion** |
| **E. 3D-тур тира** | Pannellum / Marzipano / Matterport embed | Реалистичность | Съёмка 360° или Matterport ($) | **Тур тира v2** |

#### 4.4.4. Рекомендуемая архитектура 3D (гибрид)

```
┌─────────────────────────────────────────────────────────┐
│  Устройство / prefers-reduced-motion / save-data       │
└───────────────────────────┬─────────────────────────────┘
                            │
         ┌──────────────────┼──────────────────┐
         ▼                  ▼                  ▼
   [Desktop + GPU]    [Mobile mid]      [Low-end / a11y]
   R3F интерактив    R3F упрощённый    Lottie или Video
   GLB < 2 MB        GLB < 800 KB      < 500 KB
```

**Pipeline контента:**

1. Моделирование в **Blender** (стрелок — low-poly stylized, без детализации оружия по требованиям модерации)
2. Экспорт **GLB** (Draco compression)
3. Анимации — **AnimationMixer** или пошаговые keyframes по кнопкам
4. Текстуры — **WebP**, max 1024×1024
5. Загрузка — **dynamic import**, `Suspense`, placeholder skeleton

#### 4.4.5. Производительность и доступность 3D

- 3D-блок **не блокирует** LCP: hero-poster (статичное изображение) до загрузки canvas
- При `prefers-reduced-motion: reduce` — статичный кадр или видео без autoplay
- При `navigator.connection.saveData` — только poster
- FPS-таргет: **30 fps** на mobile, **60 fps** на desktop
- Общий бюджет 3D-ассетов на страницу: **≤ 3 MB** (gzip/brotli)

#### 4.4.6. Ограничения контента 3D

- Без детализированных моделей боевого оружия в «игровом» контексте
- Стиль: **учебный / спортивный**, силуэты, схематичные мишени
- Согласование макетов 3D-сцен с заказчиком **до** финального моделирования

## 6. Функциональные требования

### 6.1. Онлайн-запись (v2)

**Поля формы:**

| Поле | Обязательность |
|------|----------------|
| ФИО | ✓ |
| Телефон | ✓ |
| Email | ✓ |
| Направление / программа | ✓ |
| Дата и время (из слотов) | ✓ |
| Комментарий | — |
| Согласие на обработку ПДн | ✓ |

**Логика:**

- Слоты синхронизированы с расписанием и лимитами мест (настраиваются в админке)
- Статусы записи: `новая` → `подтверждена` → `отменена` → `завершена`
- Email клиенту: подтверждение заявки + инструкция «что взять с собой»
- Email/Telegram администратору: новая запись
- SMS — **опционально** при подключении шлюза (SMS.ru, smsc.ru); не блокирует запуск v2
- reCAPTCHA v3 или Yandex SmartCaptcha

**v1 (до v2):** та же форма без привязки к слотам — выбор желаемой даты из dropdown + комментарий.

### 6.2. Калькулятор (v2)

- Источник цен — **админка** (не hardcode)
- Валюта: ₽, формат с пробелами: `12 500 ₽`
- Prefill записи: query-параметры `?service=...&price=...`

### 6.3. Тренажёр (v2)

- CRUD вопросов в админке: текст, 2–4 варианта, правильный, пояснение, категория
- Импорт из CSV (опционально)
- Статистика в админке: кол-во прохождений, средний балл (анонимно)

### 6.4. CMS / Админ-панель

| Модуль | v1 | v2 |
|--------|----|----|
| Страницы и блоки | ✓ | ✓ |
| Документы (upload PDF/DOCX) | ✓ | ✓ |
| FAQ | ✓ | ✓ |
| Новости | ✓ | ✓ |
| Расписание и слоты | — | ✓ |
| Записи клиентов | — | ✓ |
| Цены калькулятора | — | ✓ |
| База вопросов теста | — | ✓ |
| Пользователи админки (роли) | ✓ | ✓ |

**Роли:** `superadmin`, `content`, `operator` (только записи и расписание)

### 6.5. Версия для слабовидящих

- Кнопка в хедере
- Увеличение шрифта до 18–20px
- Высококонтрастная тема
- Сохранение в `localStorage`

### 6.6. Исключено из v1–v2
